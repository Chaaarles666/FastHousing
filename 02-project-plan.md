# 买房决策助手 — 项目总体规划

## 项目信息

- **项目名称**：买房决策助手（暂定）
- **Owner**：Charles
- **开发方式**：牛子哥做文档+规划+Prompt → Charles 用 Cursor 开发
- **技术栈**：Next.js + Tailwind CSS + shadcn/ui（前端）；Python FastAPI（后端/数据）
- **目标形态**：H5网页（移动优先），后期可封装微信小程序

---

## 里程碑规划

### Phase 1 — MVP（第1-2周）
**目标**：验证核心价值，能用就行

| 功能 | 描述 | 状态 |
|------|------|------|
| 房源对比器 | 手动添加房源信息，自定义权重打分，雷达图对比 | 待开发 |
| 交易 Checklist | 按阶段的标准流程+避坑指南 | 待开发 |
| 费用计算器 | 月供、税费、总成本 | 待开发 |

### Phase 2 — 数据驱动（第3-4周）
**目标**：接入真实数据，提升可信度

| 功能 | 描述 | 状态 |
|------|------|------|
| 价格透视镜 | 小区历史成交价走势、挂牌vs成交价差 | 待开发 |
| 数据爬虫 | 贝壳/链家数据采集pipeline | 待开发 |
| 小区情报 | 物业评分、周边配套、学区信息 | 待开发 |

### Phase 3 — 智能化（第5-8周）
**目标**：AI加持，提升体验

| 功能 | 描述 | 状态 |
|------|------|------|
| AI选房顾问 | 对话式需求采集 → 房源推荐 | 待开发 |
| 智能估价 | 基于成交数据的合理价格区间判断 | 待开发 |
| 分析报告 | 一键生成某套房的完整分析报告（可付费） | 待开发 |

### Phase 4 — 商业化（第9-12周）
**目标**：跑通营收模型

| 功能 | 描述 | 状态 |
|------|------|------|
| 用户系统 | 注册登录、收藏、历史记录 | 待开发 |
| 付费功能 | 高级报告、会员订阅 | 待开发 |
| 内容运营 | 小红书/知乎引流内容体系 | 待开发 |
