# 买房决策助手 — 市场调研报告

## MARKET RESEARCH BRIEF

```
Decision:        是否值得开发一款面向中国买房者的决策辅助工具
Target customer: 中国一二线城市首次/改善型购房者（25-45岁）
Geography:       中国大陆，聚焦一二线城市
Category:        房产科技（PropTech）— 买方工具赛道
Time horizon:    6-12个月MVP验证
Must-answer:     1. 市场有多大？ 2. 竞品格局如何？ 3. 差异化空间在哪？ 4. 数据获取是否可行？

```

## 一、市场规模

### TAM — 中国房产交易整体市场

- 2024年全国商品房销售额约 9.7万亿元（同比下降约17%）
- 二手房交易在一线城市占比已超60%，且持续上升
- 全国年均约600-800万套住宅交易（新房+二手房）
- 房产经纪服务市场规模约 3000-4000亿元

### SAM — 买方工具/决策辅助细分

- 目标用户：一二线城市有明确购房意向的人群
- 估算每年约 200-300万活跃购房者在一二线城市
- 愿意为决策工具付费的比例：参考其他工具类产品约 5-10%
- **SAM 估算：10-30万付费用户/年**

### SOM — 首年可触达

- 冷启动阶段，通过内容营销+口碑传播
- 首年目标：1-3万注册用户，1000-3000付费用户
- 按客单价 50-200元/次计算，首年营收 5-60万元
- **SOM 估算：保守10万，乐观50万营收**

> ⚠️ 市场规模数据基于公开统计+行业经验推算，置信度中等。建议后续用搜索趋势和用户访谈验证。

## 二、竞品分析

### 2.1 直接竞品（买方工具）

| 产品 | 定位 | 优势 | 劣势 |
|------|------|------|------|
| **贝壳找房** | 一站式交易平台 | 数据最全、用户量最大、VR看房 | 立场偏卖方（促成交易赚佣金），信息不对称仍存在 |
| **链家** | 线下+线上经纪 | 房源真实性高、经纪人专业 | 同样卖方立场，佣金导向 |
| **安居客** | 信息聚合平台 | 覆盖面广 | 虚假房源多、数据质量差 |
| **房天下** | 老牌房产门户 | 品牌认知度 | 产品老化、体验差 |
| **诸葛找房** | 数据分析工具 | 有估价和数据分析功能 | 用户量小、更新慢 |
| **幸福里**（字节） | 内容+交易 | 流量大、短视频入口 | 内容为主，工具属性弱 |

### 2.2 间接竞品（替代方案）

| 替代方案 | 用户行为 |
|----------|----------|
| **Excel/自建表格** | 自己做对比表，很多程序员和理性买家的选择 |
| **小红书/知乎攻略** | 搜"买房避坑"，看别人的经验贴 |
| **中介朋友** | 依赖熟人推荐，信息偏差大 |
| **业主群/论坛** | 某个小区的业主微信群、天涯/豆瓣讨论 |
| **购房顾问服务** | 付费请人帮选房，客单价通常3000-10000元 |

### 2.3 关键发现

**核心差距：中国房产市场缺少一个真正站在买方立场的工具。**

- 贝壳/链家的商业模式是卖方付佣金 → 本质是帮卖方成交
- 现有平台的"对比"功能很基础，缺乏个性化权重
- 交易流程中的坑点没人系统性整理
- 没有一个工具把"数据分析 + 决策框架 + 交易避坑"整合在一起

## 三、目标用户画像

### 核心用户：理性型购房者

```
年龄：25-40岁
职业：白领、程序员、金融从业者等知识工作者
特征：
  - 相信数据和逻辑，不想被中介忽悠
  - 会自己做Excel对比表但觉得太麻烦
  - 刷小红书/知乎看攻略但信息碎片化
  - 第一次买房或换房，流程不熟
  - 预算敏感但愿意为靠谱工具付费
痛点：
  1. 信息过载：看了50套房，脑子一团浆糊
  2. 对比困难：怎么在"近地铁"和"多一间房"之间取舍？
  3. 价格迷雾：这个价格到底贵不贵？有没有被宰？
  4. 流程焦虑：该先做什么后做什么？有哪些坑？
  5. 决策恐惧：这是人生最大的消费，不敢轻易决定
```

### 次要用户

- **改善型购房者**：有过一次经验，但市场变化快，需要更新认知
- **投资型购房者**：更关注数据分析、租售比、升值潜力
- **异地购房者**：在新加坡等海外，想买国内的房，信息差更大（Charles 你自己可能就是这个画像）

## 四、差异化定位

### "买家的军师" — 唯一站在买方立场的决策工具

与贝壳等平台的本质区别：

| 维度 | 贝壳/链家 | 买房决策助手（我们） |
|------|-----------|---------------------|
| 立场 | 卖方（赚佣金） | 买方（帮你省钱/避坑） |
| 目标 | 促成交易 | 帮你做出正确决策 |
| 数据呈现 | 展示房源 | 分析价格是否合理 |
| 对比功能 | 简单列表 | 加权打分+可视化 |
| 交易指导 | 撮合 | 全程避坑 checklist |

## 五、产品方案（MVP）

### 5.1 核心功能模块

**P0 — MVP必须有：**

1. **智能对比器**
   - 添加候选房源（手动输入或导入链接）
   - 自定义对比维度和权重（价格、面积、楼层、朝向、学区、通勤、噪音等）
   - 加权评分 + 雷达图可视化
   - 一键生成对比报告

2. **价格透视镜**
   - 输入小区名 → 显示历史成交价走势
   - 挂牌价 vs 实际成交价差距
   - 同小区同户型近期成交参考
   - 给出"这个价格合理吗？"的判断

3. **交易流程导航**
   - 按阶段分的标准 checklist：
     - 看房阶段（该问哪些问题）
     - 选房阶段（决策框架）
     - 谈价阶段（议价技巧+底价估算）
     - 贷款阶段（利率计算+银行对比）
     - 签约阶段（合同注意事项）
     - 过户阶段（流程+时间线）
     - 交房阶段（验房清单）
   - 每个步骤标注"常见坑点"

**P1 — 第二阶段：**

4. **AI 选房顾问**
   - 输入需求（预算、区域、通勤、学区等），推荐匹配房源
   - 对话式交互，追问细化需求

5. **社区情报**
   - 小区的物业评分、周边配套
   - 业主口碑（爬取点评数据）

6. **费用计算器**
   - 月供计算（公积金+商贷组合）
   - 税费明细
   - 装修预算估算
   - 总持有成本

### 5.2 技术架构

```
前端：Next.js + Tailwind CSS（H5优先，可封装小程序）
后端：Node.js / Python FastAPI
数据库：PostgreSQL + Redis缓存
数据源：
  - 贝壳开放平台 API（成交数据、房源信息）
  - 开源爬虫补充（lianjia-beike-spider）
  - 政府公开数据（网签数据）
AI层：
  - LLM API（GPT/Claude）用于对话式选房顾问
  - 简单规则引擎用于打分和推荐
部署：Vercel（前端）+ 云服务器（后端+爬虫）
```

### 5.3 商业模式

| 模式 | 描述 | 优先级 |
|------|------|--------|
| **免费+增值** | 基础对比免费，高级分析付费 | ⭐⭐⭐ 首选 |
| **单次报告** | 生成某套房的完整分析报告，收费 19-99元 | ⭐⭐⭐ |
| **会员订阅** | 月费制，不限次使用所有功能，29-49元/月 | ⭐⭐ |
| **付费咨询** | 连接专业购房顾问（抽佣） | ⭐ 后期 |

## 六、风险评估

| 风险 | 严重度 | 应对 |
|------|--------|------|
| **数据获取受限** | 🔴 高 | 优先接入贝壳开放平台；爬虫做补充；支持用户手动输入 |
| **贝壳等大平台抄袭** | 🟡 中 | 大平台立场矛盾（卖方佣金 vs 买方工具），短期内不会做；快速迭代建立用户粘性 |
| **房市持续低迷** | 🟡 中 | 下行市场买家更需要决策工具（怕买贵），反而是机会 |
| **用户获取成本高** | 🟡 中 | 内容营销（小红书/知乎/抖音）+ 口碑传播；核心功能免费拉新 |
| **法律合规** | 🟡 中 | 避免直接提供"投资建议"；爬虫数据注意版权 |
| **付费意愿不确定** | 🟡 中 | 先验证免费用户量级，再测试付费转化 |

## 七、RECOMMENDATION

### 证据支持的结论

1. **值得做**。中国房产市场虽然下行，但每年仍有数百万交易，且买方工具赛道几乎空白
2. **差异化明确**。"站在买方立场"是现有平台的结构性盲区，不是不想做，是商业模式冲突
3. **技术可行**。有成熟的开源爬虫和贝壳开放平台 API，数据获取有路径
4. **MVP 成本低**。一个人用 Cursor 开发，2-4周可以出 H5 MVP

### 仍不确定的

- 用户真实付费意愿和客单价（需要验证）
- 数据获取的长期稳定性（依赖第三方）
- 获客成本（需要实测）

### 建议下一步

1. **2周内**：出 MVP，核心功能只做"智能对比器"+ "交易流程 checklist"
2. **4周内**：接入价格数据，上线"价格透视镜"
3. **同步进行**：在小红书/知乎发买房攻略内容引流
4. **8周评估**：看用户数据决定是否加码

### 会改变建议的因素

- 如果贝壳关闭开放平台 → 数据获取成本大幅上升
- 如果首月注册 < 500人 → 需要重新思考获客策略
- 如果大平台推出类似买方工具 → 需要更强差异化或转向垂直（如学区房专用）
