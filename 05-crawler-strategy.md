# FastHousing — 爬虫策略 & 数据获取

## 总体策略

```
数据源优先级：
1. 贝壳网页端（最全，结构清晰，有成交数据）
2. 链家网页端（和贝壳数据互补）
3. 贝壳开放平台 API（合规，但需企业资质）
4. 政府公开数据（网签数据，按需补充）
```

### 爬取范围（Phase 1 聚焦上海）

| 数据类型 | 来源 | 频率 | 预估数据量 |
|----------|------|------|------------|
| 小区基础信息 | 贝壳 | 全量一次 + 月更新 | ~15,000个小区 |
| 在售二手房 | 贝壳 | 每日增量 | ~80,000-100,000套 |
| 成交记录 | 贝壳 | 每日增量 | 每月~8,000-15,000条 |
| 价格历史 | 自己聚合 | 每日快照 | 按小区维度聚合 |

## 爬虫技术方案

### 反反爬策略

贝壳的反爬比较严格，需要以下措施：

```
1. IP 代理池
   - 使用付费代理（推荐：快代理/芝麻代理/站大爷）
   - 每次请求随机切换 IP
   - 检测到封禁自动切换

2. 请求伪装
   - 随机 User-Agent（维护一个 UA 池）
   - 模拟真实浏览器 Headers（Accept, Referer, Cookie）
   - 请求间隔随机化（2-5秒）

3. Cookie 管理
   - 首次访问获取 Cookie
   - 定期刷新 Cookie
   - 维护多个 Cookie Session 轮换

4. 验证码处理
   - 遇到验证码暂停该 Session
   - 切换 IP + Cookie 重试
   - 严重时降低爬取频率

5. 限流策略
   - 白天（9-21点）低频爬取：每分钟 5-10 请求
   - 夜间（21-9点）可适当提高：每分钟 15-20 请求
   - 单 IP 每小时不超过 200 请求
```

### 贝壳网页结构分析

```
小区列表页：
https://sh.ke.com/xiaoqu/{district}/
→ 分页遍历所有小区
→ 提取：小区名、ID、均价、在售数量、建成年代、地址

小区详情页：
https://sh.ke.com/xiaoqu/{community_id}/
→ 提取：详细信息、周边配套

二手房列表页：
https://sh.ke.com/ershoufang/{community_id}/
→ 提取：在售房源列表
→ 分页遍历

二手房详情页：
https://sh.ke.com/ershoufang/{listing_id}.html
→ 提取：完整房源信息

成交记录页：
https://sh.ke.com/chengjiao/{community_id}/
→ 提取：历史成交记录
→ 分页遍历
```

### 数据清洗规则

```python
# 价格清洗
- 总价单位统一为"万元"
- 单价单位统一为"元/㎡"
- 异常值过滤：单价 < 5000 或 > 300000 标记为异常

# 面积清洗
- 统一为㎡
- 异常值过滤：< 10㎡ 或 > 1000㎡ 标记为异常

# 户型解析
- "3室2厅1卫" → rooms=3, halls=2, bathrooms=1
- 正则提取：r'(\d+)室(\d+)厅'

# 楼层解析
- "中楼层(共18层)" → floor_number=None, total_floors=18, floor_level="中"
- "6/18层" → floor_number=6, total_floors=18

# 日期解析
- "2025.03.15成交" → deal_date=2025-03-15
- 相对日期处理："3天前" → 计算绝对日期

# 去重
- 基于 beike_id 去重（UPSERT）
- 价格变动记录到 price_change 日志
```

## 爬虫执行流程

```
Phase 1: 全量初始化（首次运行，约需2-3天）
┌─────────────────────────────────┐
│ Step 1: 爬取上海所有区的小区列表 │
│ → 约 15,000 个小区              │
│ → 存入 communities 表           │
└──────────────┬──────────────────┘
               │
┌──────────────▼──────────────────┐
│ Step 2: 逐小区爬取在售房源      │
│ → 约 80,000-100,000 套          │
│ → 存入 listings 表              │
└──────────────┬──────────────────┘
               │
┌──────────────▼──────────────────┐
│ Step 3: 逐小区爬取成交记录      │
│ → 近 6 个月成交                 │
│ → 存入 transactions 表          │
└──────────────┬──────────────────┘
               │
┌──────────────▼──────────────────┐
│ Step 4: 生成价格历史快照        │
│ → 按小区聚合均价                │
│ → 存入 price_history 表         │
└─────────────────────────────────┘

Phase 2: 每日增量更新（定时任务）
┌─────────────────────────────────┐
│ 每天凌晨 2:00-6:00 执行         │
│                                 │
│ 1. 爬取新挂牌房源               │
│    → UPSERT listings            │
│    → 标记已下架的为 removed      │
│                                 │
│ 2. 爬取新成交记录               │
│    → INSERT transactions        │
│    → 关联已有 listing            │
│                                 │
│ 3. 更新小区均价                 │
│    → UPDATE communities         │
│                                 │
│ 4. 生成当日价格快照             │
│    → INSERT price_history       │
└─────────────────────────────────┘
```

## 法律与合规注意

```
⚠️ 重要提醒：

1. 爬取数据仅供个人研究和学习使用
2. 遵守 robots.txt 指示
3. 不要对服务器造成过大压力（限速）
4. 不要爬取用户个人信息（中介手机号等）
5. 如果未来商用，需要：
   - 接入贝壳开放平台 API（合规渠道）
   - 或者与数据提供商签订数据许可协议
6. 2021年后中国对爬虫的法律监管趋严
   - 《数据安全法》《个人信息保护法》
   - 建议爬取的数据不包含个人隐私信息
```
